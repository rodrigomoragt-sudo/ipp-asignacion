Un sistema completo de planificación automática de visitas que considera múltiples criterios operativos para distribuir de forma equitativa las 58,105+ clientes entre 27 supervisores.

⚡ Criterios de Priorización

El sistema clasifica automáticamente cada cliente según reglas de negocio complejas:

1
Equipo Frío (EF)
Clientes con equipo de frío - Máxima prioridad. Mínimo 1 visita en el mes. No aplica restricción de 3 meses.
2
Blindaje/BTL
Clientes de blindaje - Alta prioridad. Se planifican después de EF. Aplica regla de 3 meses móviles.
3
Desarrollar
Clientes en desarrollo - Media prioridad. Según disponibilidad. Aplica regla de 3 meses móviles.
4
Optimizar
Clientes para optimizar - Baja prioridad. Según disponibilidad. Aplica regla de 3 meses móviles.
5
Mantener
Clientes para mantener - Mínima prioridad. Relleno según disponibilidad. Aplica regla de 3 meses móviles.

📋 Reglas Operativas Implementadas

20 Visitas por Día

Cada supervisor tiene asignadas exactamente 20 visitas por día laboral, garantizando carga de trabajo equitativa.

Regla de 3 Meses Móviles

Los clientes IPP no deben tener encuesta en los últimos 3 meses. No aplica para clientes con equipo frío.

No Repetir Frecuencia

No se repite la frecuencia de visita excepto en la última semana (Lunes-Jueves) cuando es necesario.

Días de Exclusión

Se pueden excluir fechas específicas (fin de mes, vacaciones) para evitar asignaciones innecesarias.

Inicio Flexible

Las visitas comienzan entre el 1-5 del mes, típicamente el 5 para dejar margen administrativo.

Última Semana

En la última semana (LU-JU), se permite repetir frecuencia para completar la cobertura de EF.

🚀 Cómo Usar el Sistema

Opción 1: Interfaz Web (Recomendado)

1
Abrir Terminal/CMD en la carpeta del proyecto
2
Ejecutar servidor:
py server.py
3
El navegador abre automáticamente en http://localhost:8000
4
Configurar parámetros en la pestaña "Configurar Plan"
5
Generar Plan y ver cronograma interactivo
6
Exportar como Excel o CSV desde la pestaña "Exportar"

Opción 2: Línea de Comandos

1
Generar plan de Julio:
py generar_plan.py
2
Genera automáticamente:
  • plan_visitas_julio.json
  • plan_visitas_julio.xlsx

Opción 3: Menú Interactivo (Windows)

1
Hacer doble clic en START.bat
2
Seleccionar la opción deseada del menú

⚙️ Parámetros Configurables

Mes y Año

Selecciona el período a planificar. Julio 2026 es el predeterminado.

Inicio de Visitas

Día del mes (1-5) para comenzar. Recomendado: 5 para margen administrativo.

Visitas por Día

Cantidad de clientes por supervisor por día. Estándar: 20

Días de Exclusión

Fechas sin visitas (ej: 28,29,30,31 para fin de mes)

Días Sin Visita

Días de la semana sin programación (0=Lun, 6=Dom). Típico: 5,6 (Sáb,Dom)

Filtro de Supervisor

Generar plan solo para una zona específica o todas

📊 Salidas del Sistema

Excel (.xlsx)

Contiene múltiples hojas:

  • Hoja "Resumen": Totales por supervisor y segmento
  • Hojas por Zona: Cronograma diario detallado con cliente, ruta, segmento y clasificación

JSON

Estructura jerárquica con:

  • Metadatos del plan (mes, año, fecha generación)
  • Cronograma por supervisor con visitas diarias
  • Resumen de segmentos por zona

Dashboard JSON

Datos para visualización en la interfaz web:

  • Estadísticas globales
  • Resumen por supervisor
  • Cantidad de clientes EF

💡 Casos de Uso

Caso 1: Planificación Mensual Estándar

Ejecutar el sistema con parámetros por defecto para generar plan mensual completo con todos los criterios activados.

Caso 2: Plan por Zona Específica

Usar el filtro de supervisor para planificar solo para una zona en particular, útil para análisis o cambios de última hora.

Caso 3: Ajuste de Calendario

Modificar días de exclusión o inicio de visitas para adaptarse a eventos especiales como festivos o períodos de cierre.

Caso 4: Reporte Ejecutivo

Usar el dashboard para ver métricas generales y exportar resumen para presentaciones a stakeholders.
